> **Purpose:** Assess mid-level candidates capable of bridging frontend, backend, and AI layers, demonstrating system ownership and integration skills.

---

## Evaluation Categories

| Category                          | Skills/Indicators                                              | Score (0-3) |
| --------------------------------- | -------------------------------------------------------------- | ----------- |
| **System Thinking**               | Understands full-stack flows, API contracts, and failure modes |             |
| **Backend Mastery**               | Django/DRF, ORM, async tasks, API design                       |             |
| **Frontend Mastery**              | React, TypeScript, state management, UI/UX principles          |             |
| **AI Orchestration**              | Can review, audit, and integrate AI-generated code safely      |             |
| **Testing & QA**                  | Unit, integration tests; handles edge cases                    |             |
| **Security & Auth**               | Implements JWT, permissions, secure data handling              |             |
| **Communication & Documentation** | Explains technical decisions clearly, documents system         |             |
| **Collaboration & Mentorship**    | Guides juniors, code reviews, feedback culture                 |             |

---

## Score Interpretation

| Score | Meaning                                                                |
| ----- | ---------------------------------------------------------------------- |
| 0     | Not demonstrated / unsafe / incomplete                                 |
| 1     | Partial competence, requires guidance                                  |
| 2     | Fully competent, can operate independently                             |
| 3     | Exceptional, demonstrates ownership, mentors others, anticipates risks |

---

## Notes for Interviewer

* Candidates should demonstrate **ownership of end-to-end flows**, not just coding ability.
* Evaluate ability to **integrate AI responsibly** and handle system-level trade-offs.
* Include exercises that involve both **frontend + backend + AI orchestration**.
* Ask for examples of past failures and how they mitigated them.

---

> This rubric can be used for hiring mid-level Bridge Engineers, as well as for internal promotion assessments and self-audit benchmarks.
